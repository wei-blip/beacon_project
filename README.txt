СТРУКТУРА ПРОЕКТА:
    1.app_<dev_name> - папка с кодом для каждого из устройств
    2.modules - модули Zephyr OS

Для этого проекта добавлены несколько новых конфигов:
1.LORA_RR - Конфиг для использования библиотек lora_common.h и message_format.h
2.DWM_RR - Конфиг для использования библиотеки dwm_common.h
3.INDICATION_RR - Конфиг для использования библиотеки indication.h
4.CONFIG_CURRENT_DEVICE_NUM - Конфиг для с номером устройства (БС = 0)
5.CONFIG_NUMBER_OF_DEVICES - Конфиг для количества устройств в сети

Общие функции для LoRa находятся в файле lora_common.h a их описание в                                   lora_common.cc
Описание используемого протокола находится в файле message_format.h
Общие функции для DWM находятся в файлу dwm_common.h а их описание в файле dwm_common.cc
Общие функции для индикации находятся в файле indication.h а их описание в файле indication.cc

Разработанный протокол работает только для сообщений размером не превышающих 4 байта. В противном случае надо будет переписать функции для битовых операций находящихся в файле lora_common.cc
